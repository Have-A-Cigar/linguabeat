## 1. Музыкальная библиотека

### 1.1 Стартовый репертуар — советская эстрада 1940–1960-х годов

**Решение принято: 2026-06-07**

Стартовая библиотека строится на советских песнях 1940–1960-х годов.

**Обоснование:**
- **Сокращение затрат на старте.** Для произведений, где автор умер 70+ лет назад, авторские права (РАО) не действуют — произведение в публичном достоянии. Для записей, созданных до ~1975 года, истёк 50-летний срок охраны смежных прав (ВОИС). Значительная часть репертуара 1940–60-х подпадает под оба условия — лицензионные выплаты не требуются.
- **Народное достояние.** Эти песни культурно значимы, широко известны, являются частью российской идентичности — что важно для целевой аудитории (иностранные студенты, изучающие русский язык и культуру).
- **Качество педагогического материала.** Тексты литературно выверены, лексика богата, произношение в записях — эталонное.

**Правовой алгоритм проверки каждого трека:**
1. Проверить дату смерти автора (композитор + поэт) → если 70+ лет назад → авторские права свободны
2. Проверить дату создания конкретной записи → если до 1975 года → смежные права свободны
3. Если хотя бы одно условие не выполнено → заключить прямой лицензионный договор с правообладателем / наследниками

**Целевой объём:** ~500 треков для полноценной библиотеки; 50–100 треков для пилота.

**Ссылки:** см. файл `docs/soviet-songs-library.md` — список ~500 произведений.

## 2. Лицензирование музыки

### 2.1 Модель хранения аудиофайлов

**Решение принято: 2026-06-07**

Аудиофайлы хранятся на **собственном S3 / Object Storage** LinguaBeat — не стримируются с серверов правообладателя.

**Обоснование:** Это стандартная практика всех российских стримингов (Яндекс Музыка, VK Музыка). Правообладатель предоставляет лицензию и передаёт файл; место хранения юридически нейтрально при наличии лицензии.

### 2.2 Двойная лицензионная схема

**Решение принято: 2026-06-07**

Для любого трека, не находящегося в публичном достоянии, требуются **два отдельных договора**:

| Организация | Что покрывает | Сайт |
|---|---|---|
| **РАО** | Авторские права — композитор, поэт-текстовик | rao.ru |
| **ВОИС** (неофиц. «Первый Музыкальный») | Смежные права — исполнитель, лейбл, конкретная запись | rosvois.ru |

**Отчётность:** ежеквартальный отчёт в оба ОКУПа: треклист + количество прослушиваний + доход за период.

## 3. Перевод слов

### 3.1 Архитектура сервиса перевода

**Решение принято: 2026-06-07**

Принята двухуровневая схема: **Google Cloud Translation v3 (основной) + LLM fallback (контекстный)**.

```
Пользователь тапает слово
    ↓
[Кэш в БД] → если слово из этого трека уже переводили → вернуть сразу (бесплатно)
    ↓ кэш-мисс
Google Cloud Translation API v3
    → быстро, покрывает Ru→Zh, Ru→Hi, Ru→Ar, Ru→Es, Ru→En
    → 500K символов/мес бесплатно (достаточно для MVP)
    ↓ слово входит в список омонимов? (замок/коса/мир/свет/лук/...)
LLM — Gemini 2.0 Flash Lite или GPT-4o mini
    → промпт: "Переведи слово '{word}' в контексте строфы: '{lyric}'. Язык: {lang}"
    → возвращает точный перевод с учётом контекста
    ↓
Сохранить перевод в БД (не переводить одно и то же слово из одного трека дважды)
```

**Провайдеры — обоснование выбора:**

| Провайдер | Решение | Причина |
|---|---|---|
| Google Cloud Translate v3 | ✅ Основной | Все нужные языки (включая хинди), высокое качество Ru→Zh, free tier 500K/мес |
| Яндекс Translate | ⛔ Отказ | Санкционный риск для международного продукта |
| DeepL | ⛔ Отказ | Не поддерживает хинди — выпадает ключевая аудитория |
| Reverso Context | ⛔ Отказ | Нет публичного API; только неофициальный scraping — неприемлемо для production |
| LLM (Gemini Flash Lite) | ✅ Fallback | Единственный способ настоящего контекстного перевода омонимов; $0.075/млн токенов |

**Прогнозируемый бюджет:**

| Этап | Стоимость |
|---|---|
| MVP (< 500K символов/мес) | $0 (Google free tier) |
| Пилот, 100 активных пользователей | $5–15/мес |
| Масштаб, 10K MAU | $50–100/мес |

**Реализация в коде:** `backend/app/services/translation_service.py` (не создан — следующий шаг).
Дополнительно нужен список омонимов русского языка (~200 слов) как тригер LLM-пути.

## 4. Алгоритм интервального повторения

### 4.1 Миграция SM-2 → FSRS-5

**Решение принято: 2026-06-07**

Алгоритм повторения — **FSRS-5** (библиотека `fsrs` v6, Python).

**Обоснование:** FSRS обучен на 350 млн сессий Anki, даёт −25% повторений при том же retention 90% по сравнению с устаревшим SM-2 (1987).

**Реализация:** `backend/app/domain/core_srs.py` — изолированный domain-модуль без зависимостей от веб-слоя.

## 5. Контракты API

### 5.1 Contract-first разработка через OpenAPI

**Решение принято: 2026-06-07**

Единственный источник правды для API-контракта — **Pydantic-схемы** в `backend/app/schemas/`.

Workflow:
```
Pydantic-схемы → gen_openapi.py → openapi.json → @hey-api/openapi-ts → frontend/src/api/generated/
```

Файлы в `frontend/src/api/generated/` — сгенерированы автоматически, **не редактировать вручную**.
Скрипт запуска: `./gen-api.ps1`
