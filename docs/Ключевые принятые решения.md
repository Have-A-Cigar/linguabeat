## 1. Музыкальная библиотека

### 1.1 Стартовый репертуар — советская эстрада 1940–1960-х

**Решение принято: 2026-06-07**

Стартовая библиотека строится на советских песнях 1940–1960-х.

**Почему:**
- **Бесплатная лицензия.** Если автор музыки и поэт умерли 70+ лет назад — авторские права (РАО) истекли. Если запись сделана до ~1975 года — истёк 50-летний срок смежных прав (ВОИС). Большинство треков 1940–60-х попадают под оба условия, лицензионных выплат нет.
- **Культурная ценность.** Эти песни знает любой, кто изучает русский язык и культуру. Целевая аудитория — иностранные студенты в российских вузах.
- **Качество текстов.** Литературный русский, богатая лексика, чёткая артикуляция исполнителей — хорошо для обучения.

**Как проверять каждый трек:**
1. Дата смерти автора (композитор + поэт) → 70+ лет назад → авторские права свободны
2. Дата конкретной записи → до 1975 года → смежные права свободны
3. Хотя бы одно условие не выполнено → прямой лицензионный договор с правообладателем

**Объём:** ~500 треков для полноценной библиотеки, 50–100 для пилота.

Список: `docs/soviet-songs-library.md`.

## 2. Лицензирование музыки

### 2.1 Хранение аудиофайлов

**Решение принято: 2026-06-07**

Аудиофайлы лежат на собственном S3 / Object Storage LinguaBeat — не стримируются с серверов правообладателя.

Это стандарт всех российских стримингов (Яндекс Музыка, VK Музыка). Правообладатель выдаёт лицензию и передаёт файл; место хранения юридически нейтрально при действующей лицензии.

### 2.2 Двойная лицензия

**Решение принято: 2026-06-07**

Для каждого трека вне публичного достояния нужны два отдельных договора:

| Организация | Что покрывает | Сайт |
|---|---|---|
| **РАО** | Авторские права — композитор, поэт | rao.ru |
| **ВОИС** (Первый Музыкальный) | Смежные права — исполнитель, лейбл, запись | rosvois.ru |

Отчётность: ежеквартально в оба ОКУПа — треклист, количество прослушиваний, доход за период.

## 3. Перевод слов

### 3.1 Архитектура сервиса перевода

**Решение принято: 2026-06-07**

Двухуровневая схема: **Google Cloud Translation v3 (основной) + LLM fallback (контекстный)**.

```
Пользователь тапает слово
    ↓
[Кэш в БД] → слово из этого трека уже переводили → вернуть сразу (бесплатно)
    ↓ кэш-мисс
Google Cloud Translation API v3
    → Ru→Zh, Ru→Hi, Ru→Ar, Ru→Es, Ru→En
    → 500K символов/мес бесплатно (хватает на MVP)
    ↓ слово в списке омонимов? (замок/коса/мир/свет/лук/...)
LLM — Gemini 2.0 Flash Lite или GPT-4o mini
    → промпт: "Переведи слово '{word}' в контексте строфы: '{lyric}'. Язык: {lang}"
    → точный перевод с учётом контекста
    ↓
Сохранить в БД — одно слово из одного трека переводим один раз
```

**Выбор провайдеров:**

| Провайдер | Решение | Причина |
|---|---|---|
| Google Cloud Translate v3 | ✅ Основной | Все нужные языки включая хинди, free tier 500K/мес |
| Яндекс Translate | ⛔ Отказ | Санкционный риск для международного продукта |
| DeepL | ⛔ Отказ | Нет хинди — выпадает ключевая аудитория |
| Reverso Context | ⛔ Отказ | Нет публичного API, только scraping — неприемлемо |
| LLM (Gemini Flash Lite) | ✅ Fallback | Единственный способ контекстного перевода омонимов; $0.075/млн токенов |

**Бюджет:**

| Этап | Стоимость |
|---|---|
| MVP (< 500K символов/мес) | $0 (Google free tier) |
| Пилот, 100 активных пользователей | $5–15/мес |
| 10K MAU | $50–100/мес |

Реализация: `backend/app/services/translation_service.py`.
Дополнительно нужен список омонимов (~200 слов) как триггер LLM-пути.

## 4. Алгоритм интервального повторения

### 4.1 Переход SM-2 → FSRS-5

**Решение принято: 2026-06-07**

Используем **FSRS-5** (библиотека `fsrs` v6, Python).

SM-2 (1987) давно устарел. FSRS обучен на 350 млн сессий Anki — даёт на 25% меньше повторений при том же retention 90%.

Реализация: `backend/app/domain/core_srs.py` — изолированный domain-модуль без зависимостей от веб-слоя.

## 5. Контракты API

### 5.1 Contract-first через OpenAPI

**Решение принято: 2026-06-07**

Единственный источник правды для API-контракта — **Pydantic-схемы** в `backend/app/schemas/`.

```
Pydantic-схемы → gen_openapi.py → openapi.json → @hey-api/openapi-ts → frontend/src/api/generated/
```

Файлы в `frontend/src/api/generated/` генерируются автоматически — не редактировать вручную.
Скрипт: `./gen-api.ps1`
